# 21. Hardware Profiles

## 21.1 Overview

Hardware profiles are the mechanism through which Samaris OS adapts its behaviour to the capabilities of the underlying machine. Generated by the Adaptive System Configuration (ASC) subsystem at boot time, each profile defines the system's resource allocation policies, performance characteristics, and feature availability.

## 21.2 Profile Definitions

### Safe

The default fallback profile when hardware detection fails or produces unreliable results.

- **Target**: any hardware with unknown or unreliable detection
- **Workers**: 1–2, conservative scaling
- **Memory compression**: enabled, moderate level
- **GPU**: software rendering, safe mode
- **Animations**: disabled or minimal
- **Cache**: limited
- **Background tasks**: restricted

### Low RAM

Optimised for memory-constrained machines (≤2 GB RAM).

- **Target**: 2 GB RAM, 2 CPU cores, any GPU
- **Workers**: 1–2, queue-based scaling
- **Memory compression**: aggressive (ZSTD level 5 for inactive)
- **GPU**: lightweight profile, CPU fallback preferred
- **Animations**: minimal
- **Cache**: minimal (32 MiB VRM cache)
- **Thumbnails**: deferred
- **Background apps**: suspended when not in focus

### Balanced

The default profile for standard desktop hardware.

- **Target**: 4–8 GB RAM, 4+ CPU cores, integrated GPU
- **Workers**: 2–4, adaptive scaling
- **Memory compression**: enabled (ZSTD level 3)
- **GPU**: hardware accelerated, medium effects
- **Animations**: normal
- **Cache**: moderate (128 MiB)
- **Thumbnails**: on-access generation

### Performance

For high-end workstations with abundant resources.

- **Target**: 16+ GB RAM, 6+ CPU cores, dedicated GPU
- **Workers**: 4–8, performance-oriented scaling
- **Memory compression**: conservative (ZSTD level 1, primarily for cold data)
- **GPU**: premium profile, full effects
- **Animations**: full, with transitions and parallax
- **Cache**: generous (256+ MiB)
- **Thumbnails**: immediate generation
- **Multitasking**: full concurrent app support

### Powersave

For laptop and battery-powered operation.

- **Target**: any laptop hardware, any RAM
- **Workers**: reduced, thermal-aware scaling
- **Memory compression**: enabled, moderate
- **GPU**: balanced with lower clock preference
- **Animations**: reduced
- **Background tasks**: throttled on battery
- **Orbit AI**: burst disabled on battery

### VM

For virtualised environments with virtio or emulated hardware.

- **Target**: any virtual machine, any RAM
- **Workers**: 1–2 (limited by virtual cores)
- **Memory compression**: enabled, moderate
- **GPU**: CPU fallback or virtio-gpu
- **Animations**: minimal to reduced
- **Cache**: moderate
- **Virtio support**: enabled if detected

### USB Boot

For live USB boot sessions with potentially limited write endurance.

- **Target**: USB-booted system, any RAM
- **Workers**: adaptive, standard
- **Memory compression**: enabled
- **GPU**: hardware dependent
- **Cache**: RAM-preferred, minimised writes to USB
- **Journal**: enabled for crash safety
- **Persistence**: user data written to RAM with optional sync

### Debug

Development profile with verbose logging and diagnostic features.

- **Target**: development machines
- **Workers**: as detected
- **Memory compression**: configurable
- **GPU**: as detected
- **Tracing**: all components at debug log level
- **DevTools**: Electron DevTools available
- **Metrics**: full latency histograms and utilisation tracking

## 21.3 Profile Selection Logic

```
ASSUME balanced
IF VM detected       → select vm
IF low_ram           → select low_ram
IF laptop            → prefer powersave (unless AC + user override)
IF usb_boot          → select usb_boot
IF debug mode        → select debug
IF detection failed  → select safe
ELSE                 → keep balanced

Apply user overrides if present
```

## 21.4 Hardware Detection Parameters

| Dimension | Detection Sources |
|-----------|-------------------|
| CPU | /proc/cpuinfo, vendor flags, core count |
| RAM | /proc/meminfo, sysinfo() |
| GPU | /dev/dri device presence, lspci fallback, wgpu enumeration |
| VRAM | Heuristic from GPU class, driver-reported |
| Storage | /sys/block rotational flag, NVMe detection |
| Boot medium | /proc/cmdline root= parameter, udev properties |
| VM detection | CPUID hypervisor bit, dmidecode system manufacturer |
| Laptop | ACPI battery presence, chassis type |
| Network | /sys/class/net interface enumeration |
