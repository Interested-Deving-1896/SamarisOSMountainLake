# Contributing

## Documentation

This documentation is a living resource. Contributions are welcome:

- **Fix errors** — Typos, broken links, outdated information
- **Add depth** — Expand existing sections with more detail
- **New content** — Guides, tutorials, reference material

<br>

## How to Contribute

1. Fork or clone the documentation repository
2. Make your changes following the style guidelines below
3. Verify all links are functional
4. Submit a pull request or contact the maintainer

<br>

## Style Guidelines

- Use **clear, concise** language
- Include **code examples** where applicable
- Maintain consistent formatting (tables, headers, lists)
- Keep **cross-links** functional between docs
- Add `[← Back: Documentation Index](../index.md)` navigation at the bottom of each page
- Reference VOLT architecture chapters when describing system behavior

<br>

## Code Standards

- Follow existing file naming conventions (`kebab-case.md`)
- Use consistent markdown formatting (ATX headers, GFM tables, fenced code blocks)
- Keep line length under 100 characters where practical
- Validate YAML/JSON examples are syntactically correct
